FunctionsÂ¶

Definition: A function is any method for taking a list of inputs and determining the corresponding output.

Examples of functionsÂ¶

Math: We can write functions as in an algebra or calculus course, with notation like this:

  • $ f(x)=x^2-5 $

  • $ g(x,y,z)=\frac{x^2-y^2}{z} $

Which way is best?Â¶

The examples above show that you can express functions using math, English, Python, tables, and more. Although none of these ways is always better than the others, we will typically give functions names and refer to them by those names. This happens in English, math, and computer science.

RelationsÂ¶

A relation is a function whose output is always true or false, i.e., its output type is bool.

Examples of relations:

  • Any set, and things are true if you find them in the set and false if you donât. In particular:

    • A table whose rows are all the lists of inputs for which the relation is true. For example, the capitol city relation above could be represented as a table of all country-capitol pairs.

    • Every DataFrame is a relation. Every SQL table is a relation. SQL is an implementation of ârelational algebra.â (Mostly. DataFrames can have duplicate rows, and SQL tables can if we ignore the index. This is a difference between them and mathematical relations.)

  • The mathematical concept of a relation exists because it represents how humans think about a lot of things. Highlight examples from everyday language that show that we talk and think this way all the time, even in non-technical domains (like âis older thanâ and many more).

Connections between functions and relationsÂ¶

What are the similarities/differences between functions and relations?

  • On the one hand, a relation is a special kind of function; its outputs canât be just anything; they must be either true or false only. (As in our definitions.)

  • On the other hand, a function is a special kind of relation: because we restrict it so that each \(x\) relates to exactly one \(y\), you can âapplyâ \(f\) and get a single answer.

  • You can âapplyâ a relation like a function if you donât mind getting multiple outputs; you just canât write this inside a mathematical expression without it getting confusing, so we donât.

Letâs get technical for a second:

  • A function is a relation in which for each input, there is exactly one output.

  • Point out the earlier example of an invertible function, and that it simply means swapping two columns in the data table that defines the function.

  • In other words, an invertible function is a relation thatâs also a function when you swap inputs and outputs; a function is invertible if for each output, there is exactly one input.
